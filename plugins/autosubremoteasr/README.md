# AI字幕ASR

使用线上 ASR 接口从视频音轨生成带时间轴的原文字幕，并通过当前插件的接口配置翻译中文字幕。

该插件是独立新插件，不依赖本地 faster-whisper，不下载本地识别模型，语音识别只走线上接口。

## 功能特点

- 支持外挂字幕、内嵌字幕、音轨 ASR 的字幕提取优先级
- 处理前会抽样检测硬字幕，检测到烧录字幕时按“已存在字幕”跳过
- 音轨识别使用远程 `/v1/audio/transcriptions` 接口
- 默认 ASR 模型为 `whisper-1`，需要接口支持 `verbose_json` 和 `segments`
- 音频按分钟分段提取为 24kHz 单声道 MP3，分段关闭后立即上传 ASR，识别完成后删除临时分段
- 处理过程中会随机抽检音频分段和 ASR 返回时间轴
- auto 语言模式会先抽取全局音频小段探测语言，支持本地静音过滤、3 个高质量样本提前锁定和缓存复用
- ASR 和翻译接口由当前插件独立配置
- 支持使用大模型将原文字幕翻译成中文字幕
- 原始字幕语言已是中文时自动跳过翻译，不生成额外的 `zh.机翻.srt`
- 支持定时扫描媒体路径，新完整视频自动加入队列
- 支持任务队列、后台运行和进度页刷新
- 任务页按正在处理、待处理、待检测、已完成、已失败分页签展示；已完成里包含已生成字幕、已存在字幕和同语言无需翻译结果，并保留具体状态细节
- 单独适配移动端样式：配置页压缩间距，操作按钮自动换行，任务记录在手机上以卡片方式展示
- 配置页按基础开关、扫描与队列、字幕策略、接口配置和翻译参数重新分区，任务页提供后台扫描、失败重试和历史清理按钮
- ASR 接口请求期间会强制写入等待秒数，避免长请求时页面进度看起来不刷新
- 支持 ASR 分段和字幕翻译断点续跑，意外中断后复用已成功的部分
- 批量翻译使用 JSON 编号回填，并在接口返回异常时自动拆分批次重试
- 支持配置翻译接口并发数，默认 1，可按接口稳定性调高
- 视频不完整时进入等待状态，到期后重新检查，不计为失败

## 配置说明

### 基础开关

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| 启用插件 | 是否启用插件 | 否 |
| 定时扫描媒体路径 | 按 cron 周期扫描媒体路径 | 是 |
| 媒体入库自动执行 | 监听媒体入库事件并加入队列 | 是 |
| 发送通知 | 是否发送任务执行通知 | 否 |

### 扫描与队列

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| 媒体路径 | 手动扫描和定时扫描共用的文件或目录，每行一个 | 空 |
| 扫描周期(cron) | 定时扫描周期 | `*/10 * * * *` |
| 不完整重试(分钟) | 文件不完整时等待后重试 | 10 |
| 并行任务数 | 同时处理任务数量，接口不稳定时建议谨慎调高 | 1 |
| 完整解码校验 | 使用 ffmpeg 完整解码扫描视频，高 CPU，默认关闭 | 否 |

任务页提供一次性操作按钮：立即扫描、重试失败、清理已完成、清理历史。这些操作不再作为常驻开关显示在配置页。

### 字幕策略

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| 无字幕时调用ASR | 无外挂、内嵌或硬字幕时允许调用 ASR | 是 |
| 翻译成中文 | 生成非中文字幕后翻译中文字幕 | 是 |
| 自动检测语言 | 不使用音轨元数据，由 ASR 接口识别语言 | 是 |
| 字幕源偏好 | 匹配已有字幕和音轨语言时的优先级 | 英文优先 |

| 音频分段分钟 | 每段音频上传时长，范围 5-30，建议 10-15 | 10 |
| ASR超时秒 | 单段 ASR 请求超过此时间会放弃本次请求并按重试策略重新请求 | 300 |
| 接口重试次数 | ASR 和翻译请求失败时的重试次数 | 3 |
| ASR提示词 | 传给音频转写接口，用于固定原文转写风格 | 保留原文、不翻译、不改写 |

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |

| API密钥 | 接口密钥 | 空 |
| 翻译模型 | 翻译中文字幕使用的大模型 | `gpt-5-chat-latest` |
| 翻译超时秒 | 单次翻译请求超过此时间会放弃本次请求并按重试策略重新请求 | 120 |
| 使用代理服务器 | 是否使用 MoviePilot 代理访问接口 | 否 |
| 详细接口日志 | 记录完整接口入参和返回，日志量很大，排查问题时再开启 | 否 |

### 翻译参数

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| 翻译接口并发数 | 同时发送的翻译批次数，接口不稳定时保持 1 | 1 |
| 每批翻译行数 | 每批处理的字幕行数，范围 1-50，建议 20-25 | 20 |
| 上下文窗口大小 | 翻译时参考的上下文行数 | 5 |
| 翻译英文时合并整句 | 英文字幕先合并整句再翻译 | 否 |

## 字幕提取策略

字幕提取优先级：外挂字幕 > 内嵌字幕 > 硬字幕检测 > 接口 ASR。

硬字幕检测会在视频完整性通过后抽取多个避开片头片尾的画面帧，分析底部字幕区域的边缘密度、文字形连通块、横向覆盖、行高范围和中部画面对照差异。只有多帧连续命中且平均置信度达到阈值时才会判定为硬字幕，尽量避免把复杂画面误判成字幕。

音轨 ASR 只负责生成原文字幕和时间轴；中文字幕由翻译流程生成。这样比直接让音频大模型输出中文字幕更稳定，时间轴也更可控。

如果原字幕或 ASR 识别结果已经是中文，插件会保留原始中文字幕并跳过翻译接口，不再生成额外的机翻字幕文件。

音频处理采用流水线方式：ffmpeg 写完一个分段后，插件立即上传该分段到 ASR 接口，识别完成后删除临时音频文件。运行中只会保留正在写入的最后一个分段和少量待处理分段，不会等待整部视频全部提取完成后再开始识别。

## 注意事项

1. ASR 接口必须支持 `/v1/audio/transcriptions`、`verbose_json` 和 `segments`，否则无法稳定生成 SRT 时间轴。
2. 当前线上测试可用的稳定 ASR 模型为 `whisper-1`。
3. 插件不会下载或加载本地语音识别模型。
4. 自动检测语言时会优先复用同一视频、同一音轨、同一模型的探测缓存，视频大小或修改时间变化后缓存自动失效。
5. 随机抽检使用 ffprobe 读取分段元数据，并检查 ASR 返回的 segments 时间轴是否递增。
6. 定时扫描不会重复触发已完成、已忽略、处理中或已在队列中的视频。
7. 翻译全部失败时不会标记任务完成，也不会生成全是“翻译失败”的中文字幕。
8. 批量翻译会要求模型返回带 id 的 JSON；如果模型未按格式返回，插件会自动拆分批次重试，最后才退到单行。
9. ffmpeg 相关处理固定使用 1 个线程，降低 CPU 峰值。
10. 硬字幕检测是保守策略，误判为已有字幕会跳过处理；如遇个别视频误判，可清理对应任务记录后再处理。
11. 中文语言识别依赖字幕流或 ASR 返回的语言标记，支持 `zh/chi/zho/chs/zhs/zh-CN` 等常见标记。
12. 插件关闭时会中断当前 ffmpeg 音频提取；接口请求会在超时或返回后结束。
