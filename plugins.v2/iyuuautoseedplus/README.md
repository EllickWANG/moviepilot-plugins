# IYUU自动辅种增强

插件 ID：`IYUUAutoSeedPlus`

## 用途

本插件复刻官方 `IYUUAutoSeed` 2.15，并额外增加批量查询、网络超时和站点域名匹配配置，用于解决 IYUU 查询接口在大量 hash 场景下容易 20 秒超时，以及 IYUU 返回域名与 MoviePilot 统一站点配置不一致的问题。

官方插件默认每批提交 200 个 hash，并使用 MoviePilot 默认请求超时 20 秒。线上出现过以下日志：

```text
下载器 fn-main 保种 开始查询辅种，数量：200 ...
请求失败: HTTPSConnectionPool(host='2025.iyuu.cn', port=443): Read timed out. (read timeout=20)
当前种子列表没有可辅种的站点：请求IYUU失败，未获取到返回信息
```

增强版将这两个参数改为可配置：

- `IYUU请求超时(秒)`：默认 `60`，范围 `5-300`。
- `每批查询Hash数`：默认 `50`，范围 `1-200`。

同时增强版会统一处理站点下载链接拼接，避免站点地址末尾没有 `/` 时生成 `example.comdownload.php` 这类错误；需要访问站点详情页提取下载链接时，也会使用插件中的请求超时配置。

## 与官方插件的关系

- 官方插件 ID：`IYUUAutoSeed`
- 增强版插件 ID：`IYUUAutoSeedPlus`

增强版是独立插件，不覆盖官方插件。建议二选一启用，不要同时启用官方版和增强版，否则可能重复扫描同一个下载器并重复辅种。

## 安装

在 MoviePilot 插件市场添加私人仓库：

```text
https://github.com/EllickWANG/moviepilot-plugins
```

搜索并安装：

```text
IYUU自动辅种增强
```

安装后建议停用官方 `IYUU自动辅种`，只保留增强版。

## 配置项

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| 启用插件 | 关闭 | 打开后需要同时配置执行周期、Token 和下载器，插件运行态才会变为启用。 |
| 发送通知 | 关闭 | 辅种完成后发送站内通知。 |
| 清除缓存后运行 | 关闭 | 清理成功、失败缓存后再执行。 |
| 立即运行一次 | 关闭 | 保存后触发一次辅种任务。 |
| IYUU Token | 空 | IYUU Token。不要提交到仓库或文档。 |
| 执行周期 | 空 | 标准 5 段 cron，例如 `0 1 * * *`。 |
| IYUU请求超时(秒) | `60` | 调用 `2025.iyuu.cn` 的请求超时时间。 |
| 每批查询Hash数 | `50` | 每次提交给 IYUU 查询的 hash 数量。 |
| 站点域名别名 | 空 | 当 IYUU 返回域名和 MoviePilot 站点域名不一致时配置，一行一个 `IYUU域名=MoviePilot站点域名`。 |
| 下载器 | 空 | 要扫描的主下载器，可多选。 |
| 主辅分离 | 空 | 可指定单独辅种下载器。 |
| 辅种体积大于(GB) | 空 | 小于该体积的种子跳过。 |
| 辅种站点 | 空 | 只对选定站点辅种。 |
| 不辅种标签 | 空 | 命中标签的原种跳过，多个用英文逗号分隔。 |
| 辅种后增加标签 | `已整理,辅种` | 辅种成功后给新种子增加标签。 |
| 辅种后增加分类 | 空 | 给新种子设置分类。 |
| 不辅种数据文件目录 | 空 | 保存路径命中这些目录时跳过，一行一个。 |
| 将站点名添加到标签中 | 关闭 | 给辅种任务追加站点名标签。 |
| 跳过校验(仅QB有效) | 关闭 | qBittorrent 添加任务时跳过校验。谨慎开启。 |
| 自动开始(跳过校验有效) | 关闭 | 跳过校验时自动开始任务。 |
| 分类复用(仅QB有效) | 关闭 | 复用原种分类；原种无分类时使用“辅种后增加分类”。 |

## 推荐配置

当前线上 `fn-main 保种` 已完成种子数量在几百级别时，推荐先用：

```text
IYUU请求超时(秒)：60
每批查询Hash数：50
执行周期：0 1 * * *
```

如果仍然超时：

```text
每批查询Hash数：20
IYUU请求超时(秒)：90
```

如果 IYUU 返回很快，可以逐步调大每批数量，例如 `80` 或 `100`。

如果日志里出现 `没有维护种子对应的站点：www.example.com`，但 MoviePilot 站点管理中实际已经有该站点，可以在“站点域名别名”中增加：

```text
www.example.com=example.com
```

也支持把 IYUU 返回的子域名映射到站点管理中的主域名，例如：

```text
na.example.com=example.com
```

不要使用：

```text
* 1 * * *
```

这代表每天 `1:00-1:59` 每分钟运行一次，会在一小时内跑 60 次。每天 1 点跑一次应写：

```text
0 1 * * *
```

## 执行逻辑

1. 扫描配置中的下载器。
2. 读取已完成种子。
3. 按标签、路径、体积过滤。
4. 按“每批查询Hash数”切分 hash。
5. 调用 IYUU 查询可辅种站点。
6. 按站点域名、去 `www` 域名、别名配置和子域名兜底匹配 MoviePilot 站点配置。
7. 对可辅种结果下载种子并添加到下载器。
8. 根据配置设置标签、分类、跳过校验、自动开始。
9. 保存成功、失败缓存。

## 常见问题

### 插件打开后运行态仍然是关闭

增强版保留官方判断逻辑，必须同时满足：

- 已启用插件
- 已填写执行周期
- 已填写 IYUU Token
- 已选择下载器

缺任意一项，运行态都会是关闭。

### 仍然提示 IYUU 超时

先把“每批查询Hash数”降到 `20`，并把“IYUU请求超时(秒)”提高到 `90`。如果仍然失败，多半是 IYUU 当前服务端繁忙或网络链路抖动。

### 提示没有可辅种站点

需要区分两种情况：

- `IYUU查询超时或网络异常`：这是接口超时，不代表真的没有可辅种。
- `当前种子列表没有可辅种的站点` 且返回 IYUU 状态码：这是 IYUU 正常响应后的结果。

### 提示没有维护种子对应的站点

这通常不是 IYUU 没有数据，而是 IYUU 返回的站点域名和 MoviePilot 站点管理中维护的域名对不上。增强版会自动尝试：

- 原始域名匹配。
- 去掉 `www.` 后匹配。
- 站点域名别名匹配。
- 子域名匹配主域名。

仍然匹配不到时，把日志中的域名按 `IYUU域名=MoviePilot站点域名` 写入“站点域名别名”。

### 是否会影响官方插件

不会。增强版使用独立插件 ID 和独立配置 key。建议停用官方版，避免重复扫描和重复添加。
