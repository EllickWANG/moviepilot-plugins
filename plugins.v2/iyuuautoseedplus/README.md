# IYUU自动辅种增强

插件 ID：`IYUUAutoSeedPlus`

## 用途

本插件复刻官方 `IYUUAutoSeed` 2.15，并额外增加批量查询、IYUU 请求超时、下载器请求超时和站点域名匹配配置，用于解决 IYUU 查询接口在大量 hash 场景下容易 20 秒超时、qBittorrent Web API 偶发卡住，以及 IYUU 返回域名与 MoviePilot 统一站点配置不一致的问题。增强版也会避免同一实例重复执行辅种任务，并在插件重载时通知旧任务尽快停止，避免旧配置继续跑。

官方插件默认每批提交 200 个 hash，并使用 MoviePilot 默认请求超时 20 秒。线上出现过以下日志：

```text
下载器 fn-main 保种 开始查询辅种，数量：200 ...
请求失败: HTTPSConnectionPool(host='2025.iyuu.cn', port=443): Read timed out. (read timeout=20)
当前种子列表没有可辅种的站点：请求IYUU失败，未获取到返回信息
```

增强版将这些参数改为可配置：

- `IYUU请求超时(秒)`：默认 `60`，范围 `5-300`。
- `下载器请求超时(秒)`：默认 `15`，范围 `3-300`，用于 qBittorrent API 查询、添加种子、标签等请求。
- `每批查询Hash数`：默认 `50`，范围 `1-200`。

同时增强版会统一处理站点下载链接拼接，避免站点地址末尾没有 `/` 时生成 `example.comdownload.php` 这类错误；需要访问站点详情页提取下载链接时，也会使用插件中的请求超时配置。

## 与官方插件的关系

- 官方插件 ID：`IYUUAutoSeed`
- 增强版插件 ID：`IYUUAutoSeedPlus`

增强版是独立插件，不覆盖官方插件。建议二选一启用，不要同时启用官方版和增强版，否则可能重复扫描同一个下载器并重复辅种。

## 安装

在 MoviePilot 插件市场添加私人仓库：

```text
https://github.com/EllickWANG/moviepilot-plugins
```

搜索并安装：

```text
IYUU自动辅种增强
```

安装后建议停用官方 `IYUU自动辅种`，只保留增强版。

## 配置项

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| 启用插件 | 关闭 | 打开后需要同时配置执行周期、Token 和下载器，插件运行态才会变为启用。 |
| 发送通知 | 关闭 | 辅种完成后发送站内通知。 |
| 清除缓存后运行 | 关闭 | 清理成功、失败缓存后再执行。 |
| 立即运行一次 | 关闭 | 保存后触发一次辅种任务。 |
| IYUU Token | 空 | IYUU Token。不要提交到仓库或文档。 |
| 执行周期 | 空 | 标准 5 段 cron，例如 `0 1 * * *`。 |
| IYUU请求超时(秒) | `60` | 调用 `2025.iyuu.cn` 的请求超时时间。 |
| 下载器请求超时(秒) | `15` | 调用 qBittorrent Web API 的请求超时时间。调低可快速失败，调高可容忍慢反代。 |
| 每批查询Hash数 | `50` | 每次提交给 IYUU 查询的 hash 数量。 |
| 站点域名别名 | 空 | 当 IYUU 返回域名和 MoviePilot 站点域名不一致时配置，一行一个 `IYUU域名=MoviePilot站点域名`。 |
| 下载器 | 空 | 要扫描的主下载器，可多选。 |
| 主辅分离 | 空 | 可指定单独辅种下载器。 |
| 辅种体积大于(GB) | 空 | 小于该体积的种子跳过。 |
| 辅种站点 | 空 | 只对选定站点辅种。 |
| 不辅种标签 | 空 | 命中标签的原种跳过，多个用英文逗号分隔。 |
| 辅种后增加标签 | `已整理,辅种` | 辅种成功后给新种子增加标签。 |
| 辅种后增加分类 | 空 | 给新种子设置分类；保存路径仍使用原种真实保存路径，不使用分类默认路径。 |
| 不辅种数据文件目录 | 空 | 保存路径命中这些目录时跳过，一行一个。 |
| 将站点名添加到标签中 | 关闭 | 给辅种任务追加站点名标签。 |
| 跳过校验(仅QB有效) | 关闭 | qBittorrent 添加任务时跳过校验。谨慎开启。 |
| 自动开始(跳过校验有效) | 关闭 | 跳过校验时自动开始任务。 |

## 推荐配置

当前线上 `fn-main 保种` 已完成种子数量在几百级别时，推荐先用：

```text
IYUU请求超时(秒)：60
下载器请求超时(秒)：15
每批查询Hash数：20
执行周期：*/30 * * * *
```

如果仍然超时：

```text
每批查询Hash数：20
IYUU请求超时(秒)：90
```

如果 qBittorrent 经常 `ReadTimeout` 或 `Forbidden`，优先把 MoviePilot 下载器地址改成 qB 内网地址；仍需要走反代时，可以用“下载器请求超时(秒)”控制等待时间。想减少单轮任务占用时间可设为 `8-10`，想容忍慢链路可设为 `30-60`。

如果 IYUU 返回很快，可以逐步调大每批数量，例如 `80` 或 `100`。

如果日志里出现 `没有维护种子对应的站点：www.example.com`，但 MoviePilot 站点管理中实际已经有该站点，可以在“站点域名别名”中增加：

```text
www.example.com=example.com
```

也支持把 IYUU 返回的子域名映射到站点管理中的主域名，例如：

```text
na.example.com=example.com
```

不要使用：

```text
* 1 * * *
```

这代表每天 `1:00-1:59` 每分钟运行一次，会在一小时内跑 60 次。每天 1 点跑一次应写：

```text
0 1 * * *
```

## 执行逻辑

1. 扫描配置中的下载器。
2. 每轮任务开始前重新读取插件配置。
3. 读取已完成种子。
4. 按标签、路径、体积过滤。
5. 按“每批查询Hash数”切分 hash。
6. 调用 IYUU 查询可辅种站点。
7. 按站点域名、去 `www` 域名、别名配置和子域名兜底匹配 MoviePilot 站点配置。
8. 对可辅种结果下载种子并添加到下载器。
9. 根据配置设置标签、固定分类、跳过校验、自动开始；保存路径始终使用原种真实保存路径。
10. 保存成功、失败缓存。

批次之间是相互独立的。比如 `每批查询Hash数=20`，本次需要查询 `100` 个 hash，会拆成 5 批。如果第 1 批因为 IYUU 超时失败，只会放弃第 1 批本次结果，不会把这些 hash 写入失败缓存；后面的第 2 到第 5 批会继续执行。下一次任务仍会重新尝试第 1 批。只有插件被停用、重载或服务停止时，才会在当前请求返回后停止后续批次。

## 常见问题

### 插件打开后运行态仍然是关闭

增强版保留官方判断逻辑，必须同时满足：

- 已启用插件
- 已填写执行周期
- 已填写 IYUU Token
- 已选择下载器

缺任意一项，运行态都会是关闭。

### 仍然提示 IYUU 超时

先把“每批查询Hash数”降到 `20`，并把“IYUU请求超时(秒)”提高到 `90`。如果仍然失败，多半是 IYUU 当前服务端繁忙或网络链路抖动。

### 提示站点哈希值 require

这是 IYUU 的站点哈希 `sid_sha1` 没有初始化成功，通常发生在 `/reseed/sites/index` 或 `/reseed/sites/reportExisting` 超时后。增强版会在拿不到 `sid_sha1` 时跳过本批，不再继续发送无效查询；下一批或下一轮任务会重新尝试初始化。

### qBittorrent 经常 ReadTimeout 或 Forbidden

`IYUU请求超时(秒)` 不控制 qB。qB 相关请求由“下载器请求超时(秒)”控制。插件会把该配置写入当前 qB 客户端的请求参数，并在查询目标 hash 失败时跳过本次添加，避免无法确认是否已存在时重复添加。

如果错误仍然频繁，根因通常在 qB Web API 链路：公网反代、Host 校验、登录态、Cookie/API Key、qB WebUI 封禁或 qB 本身响应慢。优先使用 Docker 内网或局域网地址直连 qB。

### 提示没有可辅种站点

需要区分两种情况：

- `IYUU查询超时或网络异常`：这是接口超时，不代表真的没有可辅种。
- `当前种子列表没有可辅种的站点` 且返回 IYUU 状态码：这是 IYUU 正常响应后的结果。

### 提示没有维护种子对应的站点

这通常不是 IYUU 没有数据，而是 IYUU 返回的站点域名和 MoviePilot 站点管理中维护的域名对不上。增强版会自动尝试：

- 原始域名匹配。
- 去掉 `www.` 后匹配。
- 站点域名别名匹配。
- 子域名匹配主域名。

仍然匹配不到时，把日志中的域名按 `IYUU域名=MoviePilot站点域名` 写入“站点域名别名”。

### 是否会影响官方插件

不会。增强版使用独立插件 ID 和独立配置 key。建议停用官方版，避免重复扫描和重复添加。
